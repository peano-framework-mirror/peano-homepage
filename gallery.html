Rotating Heat Source

Simple parabolic benchmark problem with a rotating heat source in a two-dimensional domain. There are two videos on this setup available. The 2d video illustrates the dynamic h-adaptivity while there is a 3d animation that visualises the on-the-fly grid construction for the same 2d problem in the space-time domain. The latter can be used for space-time solvers.

Lattice Boltzmann

Philipp Neumann's Lattice-Boltzmann code: Flow around an obstacle on adaptive Cartesian grid. There's a larget collection of Lattice-Boltzmann screenshots, data, and videos available at this page. This collection also is more representative and up-to-date.


PeanoClaw / PeanoSWE

Several projects have coupled Peano's dynamic adaptive mesh management to solvers of hyperbolic equation systems. Some endeavours are listed here.

Researchers around Tobias Weinzierl have used Michael Bader's SWE teaching code to study performance patterns and tuning paradigms on Sandy Bridge and Xeon Phi processors. Here is a shallow water equation video that illustrates the coupling of Peano with the SWE teaching code. The whole work is summarised in this long video ( avi/ mp4/ ogg ) that also discusses the algorithmic ideas of the corresponding publication.

Kristof Unterweger's PeanoClaw couples PyClaw with Peano: Shockfront hitting a bubble of different material. This experiment is also available as video in 2d and 2.5d (i.e. computed as 2d problem but visualised 3d).

Roland Wittmann recently used the same software architecture to simulate flooding scenarios around Mekka.

Particle-in-Tree

1.0e6 particles with random speed run through the unit square with reflecting boundaries. The grid is refined such that the number of particles per cell does not exceed ten. As the particles do not interact with each other, this is a pure algorithmic demonstrator for particles in Peano. It does not comprise any Physics. There are also a 2d video and a 3d video available. Joint work of Bart Verleye and Tobias Weinzierl.

Another 3d video can be found here. It has a different colouring but also illustrates adaptive mesh refinement following a particle distribution. This one is encoded as ogv.
